Introduction to Git & GitHub

Git is a distributed version control system that helps developers track changes in their projects efficiently. It allows multiple people to collaborate on the same codebase while maintaining a history of modifications.

GitHub is a cloud-based platform built around Git that provides hosting for repositories, collaboration tools, and workflow automation features.

In this guide, you will learn how to set up Git, use essential commands, and follow best practices for version control.

Setup: Installing Git & Configuring GitHub

To start using Git, follow these steps:

1. Install Git

Download and install Git from the official website:

Download Git

After installation, verify it by running the following command:

git --version

2. Configure Git

Set up your username and email:

git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"

Check your settings:

git config --list

3. Create a GitHub Account

Go to GitHub and sign up.

4. Set Up SSH Key (Optional, Recommended)

To avoid entering your password every time, set up an SSH key:

ssh-keygen -t ed25519 -C "your-email@example.com"

Then add the key to GitHub under Settings â†’ SSH and GPG keys.

5. Clone an Existing Repository

To download a repository from GitHub:

git clone https://github.com/user/repository.git

6. Initialize a Local Repository

To start a new Git project:

git init
git add .
git commit -m "Initial commit"

Git Commands

Below are some essential Git commands that every beginner should know.

1. git init

Explanation: Initializes a new Git repository in the current directory.

git init

2. git clone

Explanation: Creates a local copy of a remote repository.

git clone [repository URL]

Usage Example:

$ git clone https://github.com/user/repo.git

3. git add

Explanation: Adds changes to the staging area.

git add [file]

4. git commit

Explanation: Saves staged changes with a message.

git commit -m "Your commit message"

5. git push

Explanation: Uploads local commits to a remote repository.

git push origin [branch]

6. git pull

Explanation: Fetches and merges updates from a remote repository.

git pull origin [branch]

7. git branch

Explanation: Lists, creates, or deletes branches.

git branch

8. git checkout

Explanation: Switches to another branch.

git checkout [branch]

9. git merge

Explanation: Merges changes from one branch into another.

git merge [branch]

Usage Example:

$ git checkout main
        $ git merge feature-branch

Tip: Resolve conflicts if Git cannot merge automatically.

10. git remote

Explanation: Manages remote repositories.

git remote -v

11. git log

Explanation: Displays commit history.

git log

12. git diff

Explanation: Shows differences between commits, branches, or working directory.

git diff

13. git reset

Explanation: Undoes commits or unstages files.

git reset [commit]

14. git stash

Explanation: Temporarily saves changes without committing.

git stash

Usage Example:

$ git stash save "Temporary work"

Tip: Use git stash pop to restore changes.
