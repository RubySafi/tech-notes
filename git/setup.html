Setup: Installing Git & Configuring GitHub

To start using Git, follow these steps:

1. Install Git

Download and install Git from the official website:

Download Git

After installation, verify that Git is installed by running the following command: 

git --version
[What does this command do?]

The command git --version displays the installed version of Git. This allows you to verify whether Git has been successfully installed on your system.

2. Configure Git

Set up your username and email:

git config --global user.name "Your Name"
git config --global user.email "your-email@example.com"
[What does this command do?]

The command git config is used to set configuration values for Git. The flag --global specifies that the configuration applies globally, meaning the setting will affect all Git repositories on your system. The general syntax is: git config --global <parameter> <value>, where parameter is the name of the configuration option you want to set, and value is the value you want to assign to it.

To confirm your settings, run:

git config --list
[What does this command do?]

The command git config --list displays all the Git configuration settings currently in use, including both global and local configurations. You can check whether your settings are properly reflected as part of the process described above.
Note: If there are many parameters, they may be displayed in multiple pages. If the display stops, press Enter to continue. To return to the usual console mode, press q to quit the pager.

3. Create a GitHub Account

Go to GitHub and sign up for an account.

SSH Key Setup (Optional)

There are multiple ways to clone a repository from GitHub to your local PC: HTTPS, SSH, and GitHub CLI. You can start with HTTPS, but setting up SSH provides a more secure and convenient way to authenticate.

This step is optional, and you can do it anytime. Feel free to set it up after gaining a deeper understanding of Git and GitHub.

4. Generate an SSH Key (Optional)

To avoid entering your password every time you interact with GitHub, generate an SSH key: 

ssh-keygen -t ed25519 -C "your-email@example.com"
[What does this command do?]

The command ssh-keygen -t ed25519 -C "your-email@example.com" is used to generate a new SSH key pair on your computer. Each time you run this command, a new key is created with a unique "seed" (private key).
The flag -t specifies the type of key to create. In this case, ed25519 is selected, which refers to a special elliptic curve that provides strong security. The name ed25519 represents the specific elliptic curve algorithm used, and this value cannot be changed to something else, such as ed34567.
The flag -C allows you to add a comment to the key. Typically, this comment is used to associate an email address with the key to help identify it later. In this example, the email address "your-email@example.com" is used as the comment.

You may be prompted with the following:

Enter file in which to save the key (~/.ssh/id_ed25519):

This is the default location where the generated key will be saved. Press Enter unless you want to choose a different location.

You may then be prompted with:

Enter passphrase for "~/.ssh/id_ed25519" (empty for no passphrase):

This step adds an extra layer of security to your SSH key. Press Enter to leave it empty and skip setting a passphrase, or enter a passphrase for added security. If you enter a passphrase, you will be asked to confirm it by entering it again.

5. Add Your SSH Key to GitHub

After generating the key, display the public key with the following command: 

cat ~/.ssh/id_ed25519.pub
[What does this command do?]

The command cat file1 file2 ... is used to concatenate and display the contents of one or more files. In this case, since only one file is specified, no concatenation occurs, and it simply displays the contents of that file.
Note: The command name cat is derived from the word "concatenate," which means to link things together.
Note: If you chose a custom directory for your SSH key in the previous step, make sure to update the file path accordingly.

Copy the output, then add the key to GitHub:

  2. Click New SSH key.
  3. Paste the copied key and give it a recognizable name (e.g., "My PC").
  4. Click Add SSH key to save it.

SSH Setup Complete!

Congratulations! You've successfully set up SSH authentication for GitHub.
