Preface

This page just explain git commands. Since hands-on practice is the best way to understand what command means, so I recommend you to skip this section, and to go to workflow. Once you understand terminology for git, this commands page change easy-to-understand.

Need to Know Commands

These are the most important Git commands for beginners.

1. git init

Explanation: Initializes a new Git repository.

git init

2. git clone

Explanation: Copies an existing repository from GitHub.

git clone [repository-url]

3. git add

Explanation: Adds changes to the staging area.

git add [file]

4. git commit

Explanation: Saves staged changes.

git commit -m "Your message"

5. git push

Explanation: Uploads commits to GitHub.

git push origin [branch]

6. git status

Explanation: Checks which files are modified or staged.

git status

Recommended to Know

These commands help manage repositories effectively.

7. git pull

Explanation: Fetches updates from GitHub.

git pull origin [branch]

8. git branch

Explanation: Lists, creates, or deletes branches.

git branch

9. git checkout

Explanation: Switches branches.

git checkout [branch]

10. git merge

Explanation: Merges branches.

git merge [branch]

Convenient but Optional Commands

These commands make Git more powerful.

11. git log

Explanation: Shows commit history.

git log

12. git diff

Explanation: Shows differences between commits/files.

git diff

13. git reset

Explanation: Undo changes in different ways.

git reset [option]

14. git stash

Explanation: Temporarily saves changes.

git stash
