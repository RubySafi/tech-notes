Preface

This page explains Git commands. In the workflow page, we covered the basic usage of Git/GitHub with VSCode. Most operations are done behind the UI, so we only need a few commands.

These UI operations can be translated into command-line operations. Here, we will quickly review the list of commands along with a brief explanation of each. We won't go into detail about branching and merging here, but we plan to explore those concepts in the next page.

In any case, let's explore what commands exist (and what functions they serve). As you read through, try running the commands and see how they work in your own environment.

Git Commands

Repository Setup Commands

Commands for setting up and configuring a Git repository before starting development. The following commands provide help with setting up Git.

  1. git init - Initialize a new Git repository.
  2. git clone <repo-url> - Copy a remote repository to your local machine.
  3. git config <*args> - Config related setting.

The git config command is mainly used for managing Git configuration settings. Common usages include:

  • git config --list - Display all Git configuration settings.
  • git config --global <parameter> <value> - Set a global configuration parameter.

Core Commit Workflow Commands

The essential commands for tracking, saving, and sharing changes in a Git repository.

  1. git add <file> - Move files from the working directory to the staging area.
  2. git commit -m "message" - Move files from the staging area to the local repository with a commit message.
  3. git push - Upload changes from the local repository to the remote repository.
  4. git pull - Retrieve the latest changes from the remote repository to your local machine.

Note that the usage of the push and pull commands requires an understanding of branches, which will be explained later.

Branching and Merging Commands

Commands for managing branches, switching contexts, and combining changes.

The following command is needed if you start to use essential notion branching and merging.

  1. git branch - List, create, or delete branches.
  2. git checkout <branch> - Switch to a different branch or restore files.
  3. git switch <branch> - Switch to a different branch or restore files.
  4. git merge <branch> - Merge changes from another branch into the current branch.
  5. git diff - Show changes between commits, branches, or the working directory.

Essential Daily Git Commands

Frequently used commands to check repository status, view history, undo changes, and sync with remote repositories.

  1. git status - Show the state of the working directory and staging area.
  2. git log - Show the commit history.
  3. git reset <commit> - Undo commits by resetting the HEAD to a specific commit.
  4. git fetch - Retrieve updates from a remote repository without merging.
  5. git remote -v - Show the list of remote repositories.

Intermediate Workflow Management Commands

Commands for handling temporary changes, undoing commits safely, and selectively applying commits.

  1. git stash - Temporarily save changes without committing.
  2. git revert <commit> - Create a new commit that undoes a previous commit.
  3. git cherry-pick <commit> - Apply a specific commit from another branch.

Advanced Repository Maintenance and Debugging Commands

  1. git tag <tagname> - Create a tag for a specific commit.
  2. git show <commit> - Display detailed information about a commit or object.
  3. git rm <file> - Remove a file from the working directory and staging area.
  4. git mv <old> <new> - Rename or move a file within the repository.
  5. git bisect start - Start a binary search to find a commit that introduced a bug.
  6. git reflog - Show a history of changes to branch references.
  7. git blame <file> - Show commit history for each line of a file.
  8. git clean -f - Remove untracked files from the working directory.
