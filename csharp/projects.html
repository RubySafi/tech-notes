Preface

On this page, we will explain how to generate C# projects such as console applications, form applications, etc. While it's possible to create these projects manually by setting up directories and writing default code, this approach can be time-consuming. Fortunately, the dotnet command provides a convenient way to automatically generate these templates. We will introduce the relevant commands and explain their contents.

  • Console Application
  • Form Application
  • Class Library
  • Solution Management
  • Shared Project

Console Application

In the following explanation, we assume that the project name is TestApp.

1. Command

Run the following commands:

  dotnet new console -o TestApp
  cd TestApp
  code .
[What does this command do?]

The command dotnet new console -o TestApp performs the following:

  • dotnet new console: Generates a new console application project.
  • -o TestApp: Specifies the output folder name as TestApp. This folder will contain the generated project files.
  • cd TestApp: Changes the working directory to the newly created project folder.
  • code .: Opens the project in Visual Studio Code (assuming VSCode is installed).

2. Edit Code

The auto-generated file Program.cs already works, but modifying it slightly can improve readability and help with future maintenance. Consider updating it as follows:

Program.cs 
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello, World!");
    }
}

While this updated version behaves identically to the original, it follows a clearer structure with consistent indentation, making future changes easier to implement.

Form Application

In the following explanation, we assume that the project name is TestFormApp.

1. Command

Run the following commands:

  dotnet new winforms -o TestFormApp
  cd TestFormApp
  code .
[What does this command do?]

The command dotnet new winforms -o TestFormApp creates a new Windows Forms application project named TestFormApp.

  • The -o option specifies the output directory. If the specified folder doesn't exist, it will be created automatically.
  • The command cd TestFormApp changes the working directory to the newly created project folder.
  • The command code . opens the project in Visual Studio Code.

2. Auto-Generated Structure

Below is the automatically generated top-level directory structure. Unlike the console application template, this template includes a more structured set of files suited for a GUI-based project. 

  \bin
  \obj
  Form1.cs
  Form1.Designer.cs
  Program.cs
  TestFormApp.csproj
  TestFormApp.csproj.user

The generated structure may vary slightly depending on your environment or future .NET updates. To ensure clarity, sample source code is provided below:

[+] Program.cs 
namespace TestFormApp;

static class Program
{
    /// 
    ///  The main entry point for the application.
    /// 
    [STAThread]
    static void Main()
    {
        // To customize application settings like DPI awareness or default fonts,
        // see https://aka.ms/applicationconfiguration.
        ApplicationConfiguration.Initialize();
        Application.Run(new Form1());
    }    
}

[+] Form1.Designer.cs 
namespace TestFormApp;

partial class Form1
{
    /// 
    ///  Required designer variable.
    /// 
    private System.ComponentModel.IContainer components = null;

    /// 
    ///  Clean up any resources being used.
    /// 
    /// 
    ///   true if managed resources should be disposed; otherwise false.
    /// 
    protected override void Dispose(bool disposing)
    {
        if (disposing && (components != null))
        {
            components.Dispose();
        }
        base.Dispose(disposing);
    }

    #region Windows Form Designer generated code

    /// 
    ///  Required method for Designer support - do not modify
    ///  the contents of this method with the code editor.
    /// 
    private void InitializeComponent()
    {
        this.components = new System.ComponentModel.Container();
        this.AutoScaleMode = System.Windows.Forms.AutoScaleMode.Font;
        this.ClientSize = new System.Drawing.Size(800, 450);
        this.Text = "Form1";
    }

    #endregion
}

[+] Form1.cs 
namespace TestFormApp;
      
public partial class Form1 : Form
{
    public Form1()
    {
        InitializeComponent();
    }
}

3. Additional Edits (Recommended)

For improved clarity and consistency in future coding, consider adding the following using directives at the top of both Form1.cs and Form1.Designer.cs: 

using System;
using System.Windows.Forms;
using System.Drawing;

Class Library

If you want to create a shared project intended for use in multiple projects, you need to prepare the shared project as a class library. Let's assume the project name is SharedProject. The process is simple â€” just run the following command: 

  dotnet new classlib -o SharedProject
[What does this command do?]

This command creates a new class library in a folder named SharedProject. Inside this folder, a SharedProject.csproj file is created, and the default namespace is set to SharedProject.

Solution Management

To efficiently manage multiple projects, creating a solution file is recommended. Visual Studio automatically generates a solution file, but in VSCode, we need to create it manually and add projects to the solution ourselves.

In this section, we'll create a solution named TestSolution with the following directory structure: 

TestSolution\
  TestSolution.sln
  ProjectA\
    ProjectA.csproj
  ProjectB\
    ProjectB.csproj

1. Create a Solution File

The following command creates a new folder named TestSolution and generates a solution file inside it. 

  dotnet new sln -o TestSolution
[What does this command do?]

The command dotnet new sln -o TestSolution performs two actions:

  • It creates a new folder named TestSolution if it doesn't already exist.
  • It generates a new solution file named TestSolution.sln inside that folder.

This is equivalent to the following steps combined: 

  mkdir TestSolution
  cd TestSolution
  dotnet new sln

The -o flag is a convenient shorthand that saves you from running these steps separately.

2. Create Projects

Create the projects you want to include in the solution. For example, to match the directory structure shown earlier: 

  cd TestSolution
  dotnet new console -o ProjectA
  dotnet new console -o ProjectB

3. Add Projects to the Solution File

To include your projects in the solution, run the following commands: 

  dotnet sln TestSolution.sln add ProjectA\ProjectA.csproj
  dotnet sln TestSolution.sln add ProjectB\ProjectB.csproj

Shared Project

If the solution contains multiple projects, it may be convenient to create a shared library that can be used across multiple projects. In this section, we'll explain how to set up a shared project named SharedProject to be used by ProjectA and ProjectB, with the following directory structure: 

TestSolution\
  TestSolution.sln
  ProjectA\
    ProjectA.csproj
  ProjectB\
    ProjectB.csproj
  SharedProject\
    SharedProject.csproj

1. Setup the Directory Structure

I will omit detailed explanations for this process, as you can follow the previous steps to set up the structure. However, to quickly set up, you can run the following commands: 

  dotnet new sln -o TestSolution
  cd TestSolution
  dotnet new console -o ProjectA
  dotnet new console -o ProjectB
  dotnet new classlib -o SharedProject
  dotnet sln TestSolution.sln add ProjectA\ProjectA.csproj
  dotnet sln TestSolution.sln add ProjectB\ProjectB.csproj
  dotnet sln TestSolution.sln add SharedProject\SharedProject.csproj

2. Add SharedProject as a Reference

You need to add SharedProject as a reference in both ProjectA and ProjectB. To add the reference in ProjectA, run the following command: 

  cd ProjectA
  dotnet add reference ../SharedProject/SharedProject.csproj
[What does this command do?]

This command adds a reference to the SharedProject library in ProjectA. This makes the classes and methods in SharedProject accessible from ProjectA.

Repeat this process for ProjectB to add the reference there as well.

3. Add the Using Directive in ProjectA and ProjectB

In both ProjectA.cs and ProjectB.cs, add the following using directive at the top of the file: 

using SharedProject;

4. (Optional) Verify All Projects by Building

Move to the current directory TestSolution and run the following command. This command builds all project files that are included in the solution file. While this step is optional, it can help verify that all references and setups are correct before continuing with development.

  dotnet build TestSolution.sln;
