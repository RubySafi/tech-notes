Welcome to the C# Guide

The purpose of this site is to assist developers transitioning from Visual Studio to VSCode. Hence, this is NOT intended for absolute beginners in C# syntax. Instead, it focuses solely on improving the C# development experience in VSCode.

If there are no restrictions preventing you from using Visual Studio, I strongly recommend developing in it. Visual Studio is highly optimized for C# development, especially for Windows Forms applications.

This site is targeted at developers who either cannot or choose not to use Visual Studio for reasons such as:

  • Visual Studio is too resource-intensive for your PC (e.g., limited memory, storage, or processing power).
  • You are a Mac/Linux user, and there is no fully supported version of Visual Studio available.
  • You require commercial use of C# but cannot use Visual Studio due to its licensing terms.

This guide aims to help users, particularly those needing a commercial-friendly setup, to develop in C# without incurring extra licensing costs.

Development Environment

To support commercial use and efficient development, this guide is based on the following setup:

  • Development on Windows 11.
  • Using VSCode as the primary IDE.
  • Not using C# Dev Kit due to its licensing terms, which are similar to Visual Studio.
  • Using only extensions licensed under MIT or similarly permissive licenses.

How to Use This Guide

Follow these sections to set up and develop C# projects in VSCode:

1. Setting Up C#

Learn how to install and configure a C# development environment in VSCode. Always review the licensing terms to ensure they meet your needs.

2. Creating C# Projects

Understand how to start a new C# project, both for console applications and Windows Forms.

3. Managing Project Files

Learn how to add and manage files manually, including required templates. Unlike Visual Studio, some files (e.g., `.designer` files) may need to be created manually.

4. Working with UI Controls

Understand how to add UI controls in a Windows Forms project. Unlike Visual Studio, this requires manual code modifications.
