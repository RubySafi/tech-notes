Initializing Git

This page provides a simple guide to initializing a Git repository, creating a .gitignore file, and optionally connecting your project to GitHub.

Step 1: Initialize the Repository

To start version control with Git, you can initialize a repository using either:

  • Command Line Method: Run the following command in your terminal:

    •   git init

  • VSCode UI Method: Click on the Source Control icon in the Activity Bar, then click Initialize Repository.

    • Step 2: Create a .gitignore File

    To prevent unwanted files from being tracked, create a file named .gitignore in the root of your project with the following content: 

      obj
  bin

    These folders are automatically generated when you build your project, so they should be excluded from version control.

    Step 3 (Optional): Connect to GitHub

    You can connect your project to GitHub in one of two ways:

  • VSCode UI Method: Click on the Source Control icon, then click Publish to GitHub or Publish to Branch and follow the prompts.
  • Command Line Method: Use the following commands: 

      git remote add origin https://github.com/your-username/your-repository.git
  git branch -M main
  git push -u origin main

    • The UI method is simpler for beginners, while the command line method gives you more flexibility if needed.

    Summary

    • Run git init or use the "Initialize Repository" button.
    • Create a .gitignore file with obj and bin inside.
    • (Optional) Connect your project to GitHub using either the UI or the command line.

    You're now ready to track your project with Git!
