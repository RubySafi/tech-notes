Publishing

This section explains how to publish your application. There are two main ways to release your app. The first is the self-contained method: although the total file size tends to be large, there is no need to install additional software. This method is beginner-friendly and ideal for users who may not have the .NET runtime installed. The second method is the framework-dependent deployment, which is smaller in size but requires the user to have the .NET runtime installed on their machine.

Self-Contained Method

1. Run the following command in your terminal:

  dotnet publish -c Release -r win-x64 --self-contained
[What does this command do?]
This command publishes your application in Release mode for the specified runtime (in this case, `win-x64`). The `--self-contained` flag ensures that all necessary dependencies, including the .NET runtime, are bundled with the application, so it can run independently of whether the .NET runtime is installed on the target machine.

2. After running the command, the following directory will be automatically generated:

bin\Release\net8.0-windows\win-x64\publish\

All of the files in this directory are necessary to run your application. You can distribute them as is, or compress them into a ZIP file for easier distribution.

Framework-Dependent Deployment (Alternative Method)

For smaller file sizes, you can publish your application in a framework-dependent manner. This means that the target machine must have the .NET runtime installed to run the application.

1. Run the following command in your terminal:

  dotnet publish -c Release -r win-x64

This command publishes your application for the `win-x64` runtime but without bundling the .NET runtime. The user will need to have the appropriate version of the .NET runtime installed on their machine to run your application.

2. The following directory will be generated:

bin\Release\net8.0-windows\win-x64\publish\

Like the self-contained method, you need to distribute all files from this directory. However, because the .NET runtime is not bundled, the file size will be smaller compared to the self-contained method.
